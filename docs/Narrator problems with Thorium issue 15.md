# Narrator problems with Thorium

Last updated March 18, 2026

## Assignment

George, Prashant and Sanjog have identified problems with Narrator using Thorium. There are several problems and we have been assigned the task of documenting the problems, replicating the problems, and finally creating videos that show the problems.

##  Thorium configuration

 In the book settings George has scrolling selected, i.e., not paginated. Reduced motion is turned on, and popup footnotes are disabled. In the settings available from the bookshelf the checkmark for a screen reader present is enabled. we are using the fundamental non-visual test book version 2.0.1  available from [epubtest.org](https://www.epubtest.org/test-books).

##Toolbar issues
In thorium, there is the Bookshelf view, and the book reading view. First we will focus on the book reading view. The interface is divided into a toolbar on the top portion of the screen, and the main content, which contains the book content.

Narrator in scan mode is showing bullets instead of buttons. You can arrow down and hear bullet followed by the item. With scan mode off, you can tab to the items and it will say button.

The toolbar area should be available in scan mode and expose the buttons. Ony when a person is inputting information, e.g., a search term or a bookmark note would you need to turn off scan mode.
